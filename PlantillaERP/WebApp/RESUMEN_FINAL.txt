💡 CÓDIGO RÁPIDO - PROTEGER UN CONTROLLER:

using UserRoles.Identity.Constants;
using UserRoles.Identity.Services;
using System.Security.Claims;

[Authorize]  // ← Esta línea es lo único requerido
public class ComprasController : Controller
{
    private readonly IPermissionService _permissionService;

    public ComprasController(IPermissionService permissionService)
    {
        _permissionService = permissionService;
    }

    public async Task<IActionResult> Index()
    {
        var userId = User.FindFirst(ClaimTypes.NameIdentifier)?.Value;
        
        // Verificación extra (opcional)
        if (!await _permissionService.UserHasModuleAccessAsync(userId, ModuleNames.Compras))
            return Forbid();

        return View();
    }
}

🔐 CAPAS DE SEGURIDAD IMPLEMENTADAS:

Nivel 1: [Authorize] 
         └─ Requiere estar logueado

Nivel 2: [Authorize(Roles="...")]
         └─ Requiere rol específico

Nivel 3: UserHasModuleAccessAsync()
         └─ Verifica acceso a módulo

Nivel 4: UserHasPermissionAsync()
         └─ Verifica permiso específico

Nivel 5: UI Condicional
         └─ Mostrar/ocultar botones según permisos

📞 TROUBLESHOOTING RÁPIDO:

Error: "PermissionService not registered"
Solución: Verifica que Program.cs tenga:
          builder.Services.AddScoped<IPermissionService, PermissionService>();

Error: "There are pending migrations"
Solución: dotnet ef database update

Error: "User cannot access module"
Solución: Verifica en BD:
          1. Usuario tiene rol
          2. Rol tiene permiso asignado
          3. Módulo existe en ModulePermissions

Error: "Compilation failed"
Solución: Asegúrate de incluir los usings correctos:
          using UserRoles.Identity.Constants;
          using UserRoles.Identity.Services;
